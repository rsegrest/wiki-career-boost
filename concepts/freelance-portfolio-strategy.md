# Freelance Acquisition & Portfolio Strategy

## Problem Statement

Rick's NASA work is sensitive/proprietary — he can't show it in interviews or on GitHub. He needs public, accessible portfolio pieces that demonstrate his React + AI + UX skills. He also needs a freelance client acquisition pipeline since he has limited recent freelancing experience and failed Toptal screening (1/3 parts completed in time limit, cannot retry).

---

## Part 1: UX as a Differentiator

Rick's UX team experience at NASA is a **significant competitive asset** that most full-stack developers don't have. Here's why it matters and how to leverage it:

### Why UX + Full-Stack + AI Is Rare

- Most full-stack devs are backend-leaning — UI is an afterthought
- Most UX designers can't code — they hand off to developers
- Most AI engineers focus on models, not user experience
- Rick sits at the intersection of all three: **he can design, build, AND integrate AI** — this is "full-stack product engineer" territory

### How to Position It

| Resume Term | Instead Of | Why |
|---|---|---|
| "Full-Stack Product Engineer" | "Senior Full-Stack Developer" | Implies UX awareness + product thinking |
| "AI UX Engineer" | "AI Engineer" | Niche title, even higher premium for roles that want it |
| "Design-Systems-Aware AI Engineer" | "Frontend Developer" | Shows you understand component design + AI |

### Where UX Matters Most

1. **AI-native startups** building user-facing AI products — they desperately need engineers who understand both LLM orchestration AND user experience (most AI engineers build ugly demos)
2. **Enterprise AI features** — companies adding AI to existing products (Notion, Linear, Atlassian) need someone who can integrate AI without breaking UX
3. **AI agent platforms** — agent UX is a new frontier; nobody knows what good agent UX looks like yet. Rick can help define it.

### Talking Points for Interviews

- "I've worked on a dedicated UX team — I understand the full design-to-development workflow, not just the coding part"
- "When I build AI features, I start with the user problem, not the model. Most AI engineers do it backwards."
- "I can wireframe in Figma, prototype in React, and ship to production — you don't need a separate designer for my features"
- "My UX background means my AI integrations have better user trust, better error states, and better feedback loops"

## Part 2: Portfolio Strategy

### The Constraint

NASA work = classified/proprietary. Rick needs **3-4 public projects** that demonstrate his full skill stack without touching sensitive material.

### Prioritized Build List

Based on existing portfolio ideas in [[portfolio-project-ideas]], reordered for maximum freelance-relevance:

#### Tier 1: Ship First (1-2 weekends each, immediate freelance credibility)

1. **AI-Powered UX Critique Tool** (from portfolio-ideas #1)
   - Paste a UI screenshot, LLM critiques using Nielsen heuristics + WCAG
   - Stack: React + OpenAI Vision API + shadcn/ui
   - Why first: Directly demonstrates UX + AI intersection. This IS your portfolio thesis in one app.
   - Freelance hook: "I built a tool that does UX audits — I can audit your product too"

2. **AI Resume Optimizer** (from portfolio-ideas #5)
   - Upload resume + job description, RAG suggests rewrites
   - Stack: Next.js + OpenAI + vector DB + PDF parsing
   - Why second: Shows RAG pipeline skills (the #1 freelance AI ask). Meta narrative.
   - Freelance hook: "I can build this kind of AI feature for YOUR product"

#### Tier 2: Anchor Pieces (2-4 weeks, deeper credibility)

3. **Digital Twin Simulation** (from portfolio-ideas #4)
   - Interactive simulation with AI agent optimizer
   - Stack: React canvas/SVG + state machine + LLM
   - Why: Bridges M.S. M&S + React + AI. Defens-tech companies understand "digital twin."
   - Freelance hook: Defense/aerospace clients pay premium for simulation+web+AI

4. **Open-Source Agentic Toolkit** (from portfolio-ideas #7)
   - Lightweight library for autonomous web agents
   - Stack: TypeScript + Playwright + OpenAI + orchestration
   - Why: npm package = public credibility. Agent orchestration is the hottest freelance niche.
   - Freelance hook: "I open-sourced an agent framework — I can build custom agents for your business"

### Portfolio Presentation Rules

1. **Every project gets a live demo** on Vercel/Netlify — recruiters and clients won't clone repos
2. **Every project gets a case study** (README or blog post): problem, approach, architecture, screenshots, trade-offs
3. **Every project gets a LinkedIn post** with demo GIF/video
4. **GitHub README quality matters** — bad READMEs kill good projects
5. **Add analytics** to live demos — visitor count is social proof

### Portfolio Hosting

- **GitHub** — all repos with polished READMEs
- **Personal site** (ricksegrest.dev or similar) — portfolio landing page linking to live demos
- **Vercel** — host all demos for free
- **LinkedIn Featured section** — pin top 2 projects with demo links

## Part 3: Freelance Client Acquisition

### Toptal: Closed Door, Move On

Toptal screening is brutal (3-part: language/personality, coding algorithm, live project). Failed once, can't retry. Don't waste energy on this — Toptal takes ~50% cut anyway. The platform fee means a $150/hr take-home requires a $300/hr client bill rate. You can do better direct.

### Platforms to Try (Ranked by Effort vs Reward)

| Platform | Effort | Rates | Best For | Rick's Fit |
|---|---|---|---|---|
| **Contra** | Low | $80-$200/hr, 0% platform fee | Creative/tech freelancers | HIGH — modern, no fee, good for AI work |
| **Upwork** | Medium | $60-$180/hr, 10% fee | Volume, broad client base | MEDIUM — competitive but huge market |
| **Braintrust** | Medium | $90-$180/hr, 10% fee | Enterprise clients | MEDIUM — Web3 angle, enterprise contracts |
| **Wellfound (AngelList)** | Low | Contract roles at startups | Startup contract-to-hire | HIGH — startups need React+AI contract |
| **Toptal** | CLOSED | — | — | Cannot retry |
| **Direct outreach** | High | $120-$250/hr, 0% fee | Premium clients, long-term | HIGHEST reward, most effort |

### The Acquisition Playbook (Phase 1: Weeks 1-4)

#### Week 1: Setup
- [ ] Create Contra profile — lead with "AI Engineer + UX" positioning
- [ ] Create/refresh Upwork profile — set rate at $125/hr (mid-range, will raise after first reviews)
- [ ] Create Wellfound profile — mark as "open to contract work"
- [ ] Update LinkedIn: headline to "Full-Stack AI Engineer | React + LLM Integration + UX"
- [ ] Update LinkedIn: "Open to work" (recruiters only, not public) — set to "contract" + "full-time"
- [ ] Deploy 1 portfolio piece live (AI UX Critique Tool)

#### Week 2: First Bids
- [ ] Apply to 5 Upwork jobs for React + AI integration (filter: $5K+ budget, US/UK clients)
- [ ] Post 2 Contra project offerings ("I'll add AI features to your React app", "I'll build a RAG chatbot for your docs")
- [ ] Write LinkedIn post about AI UX Critique Tool with demo link
- [ ] Identify 10 local/regional companies (Huntsville/Alabama tech scene) that might need AI integration

#### Week 3: Direct Outreach
- [ ] Reach out to 5 small SaaS companies on LinkedIn that have React frontends but no AI features
- [ ] Message: "I noticed [product] doesn't have AI features yet. I build AI integrations for SaaS products — would you be open to a 15-min chat about what's possible?"
- [ ] Reach out to 3 defense-tech companies (Anduril, Palantir, Scale AI) — not for jobs, but for contract work
- [ ] Post second portfolio piece (AI Resume Optimizer)

#### Week 4: Momentum
- [ ] Apply to 5 more Upwork jobs
- [ ] Post second LinkedIn article
- [ ] If any outreach got responses: schedule calls
- [ ] Review and adjust: what's working, what's not

### The Acquisition Playbook (Phase 2: Months 2-3)

- [ ] Ship Tier 2 portfolio pieces (Digital Twin, Agent Toolkit)
- [ ] Raise Upwork rate to $150/hr after 3-5 completed jobs
- [ ] Start saying "no" to low-budget projects
- [ ] Productize: create fixed-price packages ($5K "AI Feature Sprint", $15K "RAG Chatbot Build", $25K "Agent Integration")
- [ ] Write a case study from first freelance project
- [ ] Post case study on LinkedIn + Contra portfolio

### Alternative: Productized Services

Instead of hourly, sell outcomes:

| Package | Price | Deliverable | Time |
|---|---|---|---|
| AI Feature Sprint | $5K | Add one AI feature to existing React app (chatbot, autocomplete, smart search) | 2 weeks |
| RAG Chatbot Build | $15K | Custom chatbot trained on company docs, deployed on their site | 4 weeks |
| Agent Integration | $25K | Custom AI agent that automates a workflow (form filling, data extraction, research) | 6 weeks |
| UX + AI Audit | $3K | Audit of existing product's UX + AI feature recommendations with wireframes | 1 week |

This is how you get to $175+/hr effective rates — fixed scope, fixed price, you control efficiency.
